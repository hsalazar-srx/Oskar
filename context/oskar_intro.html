Diagram 2 of 7
OSKAR platform architecture — three modules, one foundation
Architecture
Three modules share one PostgreSQL database, one Redis instance, one auth layer, and one audit log. Two adapter patterns isolate the platform from any specific ERP (Movex today, IFS for Site B later) and any supplier API. No integration contracts between modules.
Diagram 3 of 7
Data authority — Movex remains source of truth for committed BOMs
Architecture correction
The v1.0 "platform owns the BOM" framing was corrected. Movex holds the committed production BOM — MRP, purchasing, and ISO 13485 auditors all depend on it. OSKAR holds the change workflow, draft state, version chain, and supplier intelligence: the intelligence layer Movex cannot provide.
Diagram 4 of 7
Two-site deployment — configuration, not architecture
Infrastructure
Both sites are the same SRX entity — multi-configuration, not multi-tenancy. Identical Docker images from the same repository. Site A points to Movex permanently. Site B swaps one environment variable when IFS migration begins. When Site B splits across two countries, each country gets a new Docker instance — no code changes.
Diagram 5 of 7
ECN state machine — complete workflow with error recovery
ECN module
validate_payload fires before any Movex push — pre-checking effective date conflicts, eliminating the most common class of Status 50 failures documented in the April 2018 transcript. Status 70 (Completed) is a new explicit terminal state providing the ISO 13485-compliant, immutable audit endpoint.
Diagram 6 of 7
Supplier intelligence — 13 minutes → under 90 seconds
Supplier Intelligence module
The root cause of PLMServer's abandonment: sequential API calls producing 13+ minutes per BOM. OSKAR uses asyncio.gather with per-supplier semaphores — all 100 parts fan out to all 6 suppliers simultaneously. Redis caches results for 4 hours. A mandatory Sprint 3 performance demo shows engineers sub-90-second processing with real data before UAT.

Programme

Sprint 3 includes a mandatory performance demonstration to engineers before Sprint 5 UAT. Both legacy systems have independent cutover sequences in Phase 4.
